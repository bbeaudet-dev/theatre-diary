# PRD / Spec Template

## Problem
- What issue this solves.

## Goals
- Primary outcomes this work should achieve.

## Non-goals
- Explicitly out of scope.

## Requirements
- Functional requirements.
- Technical constraints.

## Proposed approach
- Implementation outline.
- Alternatives considered.

## Acceptance criteria
- Clear, testable success conditions.

## Rollout and validation
- Testing plan.
- Release/rollout notes.
